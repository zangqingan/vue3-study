1. 概述

从模板语法里我们可以知道：vue条件渲染有真条件渲染v-if和假条件渲染v-show两种，在实际开发中可以说两者都算比较常用的。 v-if/v-else-if/v-else 其实和JavaScript里的if语句是类似的,而v-show就是css的display属性实现的。

2. v-if

v-if 指令用于条件性地渲染一块内容。这块内容只会在指令的表达式返回真值时才被渲染。
它后面可以添加 一个或多个“else if 区块”，也就是v-else-if 指令。它可以连续多次重复使用，不过要注意的是必须它必须紧跟在一个 v-if 或一个 v-else-if 元素后面。
它后面也可以只添加一个“else 区块”，也就是 v-else 指令。它只能使用一次，而且必须紧跟在一个 v-if 或者 v-else-if 元素后面，否则它将不会被识别。
v-if指令是必须依附于某个元素的，当它想要切换一组元素时，可以用template元素包裹这组元素。 然后在template元素上使用v-if即可, template元素只是一个不可见的包装器元素起到一个容器包裹作用，最后渲染的结果并不会包含这个 template 元素。v-if/v-else-if/v-else 都可以在这个元素上使用。
注意：v-if/v-else-if 指令接收的值会被做类型隐式转换,自动计算出值得真假。

3. v-show

v-show是另一个可以用来按条件显示一个元素的指令，用法基本一样。不同之处在于 v-show 会在 DOM 渲染中保留该元素；v-show 仅切换了该元素上名为 display 的 CSS 属性。也就是并没有真的移除dom。
注意：v-show 不支持在 template 元素上使用，也不能和 v-else-if/v-else 搭配使用。

4. v-if vs v-show

相同点：都能显示隐藏某个元素。
不同点：
1. v-if能在template元素上使用，也能和v-else-if/v-else 搭配使用，而v-show不能。
2. v-if是惰性的,只有初始条件为true时才会被渲染,而v-show不管初始条件如何都会被渲染
3. v-if 是“真实的”按条件渲染元素会被销毁与重建，而v-show不会只是css的display属性被切换。
4. v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。 因此，如果需要频繁切换，则使用 v-show 较好；如果在运行时绑定条件很少改变，则 v-if 会更合适。
